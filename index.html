Web Developer

I have been coding websites by hand ever since I was about 13, and I currently own and run several websites.

I'll be the first to admit that I'm not the greatest web dev in the world, but I can also say that my sites are fast, they look good, and they utilise so little JavaScript that they don't really rely on it. This is good because JS can be used for shady stuff just as easily as it can be used for perfectly legitimate stuff, so many users prefer to control what parts of sites are running JS code, or outright disable it across the board.

My websites only utilise it for basic, and not very necessary functions, such as making scroll to top buttons work properly, making navigation menus turn into dropdowns when the screen width goes below a certain level, or auto-redirecting to the mobile version of a site when it detects the user is on a phone.

If you were to turn off JS on my sites, the only things you'd lose are small quality of life improvements, the site as a whole would still work fine and you could still read everything on them without issue.
